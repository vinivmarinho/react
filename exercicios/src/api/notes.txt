1- Consumindo API no React (forma clássica):
    Hooks usados:
        - useState => Armazena dados
        - useEffect => Executa a fetch. Criando um useEffect: useEffect(() => {"código vem aqui dentro"}, [])
    Exemplo:
    
    import {useEffect, useState} from "react"

    function App() {
        const [data, setData] = useState(null)
        const [loading, setLoading] = useState(true)
        const [error, setError] = useState(null)

        useEffect(() => {
            fetch(url)
                .then(respostaDoFetch => respostaDoFetch.json())
                .then(dadosConvertidos => setData(dadosConvertidos))
                .catch(err => setError(err))
                .finally(() => setLoading(false))
        }, [])

        if(loading) return <p>Loading...</p>
        if (error) return <p>Error</p>
        return <div>{JSON.stringify(data)}</div>
    }
    - .then(response => response.json()) => Converte a resposta da API(JSON) em um objeto JS
    - .then(data => setData(data)) => Pega os dados retornados pela API e salva esses dados no estado data usando a função setData. O "data" dentro do .then é o valor retornado pela API. A função "setData(data)" atualiza o estado data do React com esse valor, fazendo o componente renderizar novamente com os novos Dados
    - .catch(err => setError(err)) => Se der erro, o "setError" atualiza o estado "error"
    - .finally(() => setLoading(false)) => Executa sempre, com erro ou sem erro, o loading vira false, ou seja, finaliza o carregamento
    - [] no final do useEffect => Dependências vazias, o useEffect roda apenas uma vez
    - return <div>{JSON.stringify(data)}</div> => Mostra os dados na tela. O JSON.stringify" é para debug/visualização, em app real é renderizado em campos específicos




2- Estados importantes em Data Fetching:
    - Loading => Enquanto espera resposta
    - Error => Se algo falhar
    - Data => Dados recebidos

3- Evitar usar fetch direto em um componente
    - A documentação do React alerta que:
        - Pode gerar waterfall de requisições
        - Pode causar lentidão
    - Para apps maiores, recomenda-se:
        - Prefetch
        - Cache
        - Bibliotecas especializadas
    
4- O ".then" é um método de um objeto chamado "Promise"


Fonte => https://react.dev/reference/react/useEffect#usage
5- O que é o useEffect => É um hook que permite a sincronização de um componente com um sistema externo. Parâmetros do useEffect: useEffect(setup, dependencies)
    - Setup => Função com a lógica do useEffect. Quando o componente é "commitado", o React executa a função setup.
    - dependencies => Dizem ao React quando o useEffect deve rodar de novo. Podem ser props, states, variáveis, funções. Se houver Dependências, o effect roda quando as Dependências mudam, Exemplo:
        - useEffect(() => {
            // setup
        }, [a, b]);
    - Se o array estiver vazio, o effect roda uma única vez
    - Passando Dependências específicas => O Effect roda
